대부분은 **위지윅 에디터**로 편하게 쓰면 돼요. 다만 아래 몇 가지는 정해진 문법을 직접 타이핑해야 합니다. 에디터에는 글자 그대로 보여도, **게시하면 예쁘게 바뀌니** 걱정 말고 그대로 쓰면 돼요.

## 후리가나 · 병음 (한자 위에 읽는 법)

`{한자|읽기}` 형식으로 쓰면 글자 위에 작게 읽는 법이 올라가요.

- 일본어 후리가나 — 입력 `{漢字|かんじ}` → 결과 {漢字|かんじ}
- 중국어 병음 — 입력 `{中文|zhōngwén}` → 결과 {中文|zhōngwén}

가운데 세로 막대 `|` 는 키보드에서 **Shift + ₩(역슬래시 `\`) 키**로 입력해요. 여러 개를 이어 쓸 수도 있어요: 입력 `{勉強|べんきょう}` → {勉強|べんきょう}.

## 코드 넣기 (기술 글)

코드는 백틱(``` ` ```) **세 개**로 위아래를 감싸면 색이 입혀지고, 마우스를 올리면 **복사 버튼**이 나와요. 첫 줄에 `title="파일이름"` 을 붙이면 파일명 탭도 생겨요.

```js title="예시.js"
const hello = "world";
console.log(hello);
```

문장 속의 짧은 코드는 백틱 하나로 감싸요: 입력 `` `SELECT * FROM users` `` → `SELECT * FROM users`.

## 이미지 첨부

에디터 툴바의 **이미지 버튼(Browse)** 으로 파일을 고르면 자동으로 글에 붙고, 게시할 때 알아서 가볍게 최적화돼요. (끌어다 놓기도 되지만 Browse 가 가장 안정적이에요.)

## 비공개(초안)로 두기

새 글은 기본이 **초안(비공개)** 이에요. 다 쓰고 오른쪽의 **"초안" 체크를 해제**하면 그때 사이트에 공개됩니다. 편하게 쓰다가 준비되면 공개하세요.

## 채워야 하는 것

제목 · slug(주소) · 날짜 · 저자 · 카테고리만 채우면 나머지 본문은 위지윅으로 자유롭게. slug 는 글 주소가 되는 값이라 **공백 대신 하이픈(-)** 을 써요. 예: `놀러-가고-싶다`.
