글 쓰는 순서대로 따라오면 돼요. 대부분은 **위지윅(보이는 그대로) 에디터**로 편하게 쓰고, 몇 가지 특별한 것만 정해진 문법을 씁니다.

## 1. 로그인 (GitHub 계정 필요)

글은 **관리자 페이지**에서 씁니다.

1. 주소창에 **`livvy.byb.kr/admin`** 을 입력해 들어가요.
2. **“GitHub으로 로그인”** 버튼을 누르고, GitHub 로그인 → 처음이면 접근 **승인(Authorize)** 을 눌러요.
3. **GitHub 계정이 없으면** 먼저 [github.com](https://github.com) 에서 무료로 가입해야 해요. (가입 후 용배에게 말하면 편집 권한을 추가해줘요.)

> 로그인이 안 되거나 권한 오류가 나면 용배에게 알려주세요.

## 2. 로그인하면 보이는 화면

- 가운데에 **지금까지 쓴 글 목록**이 보여요.
- 왼쪽/위에 **“글”, “페이지”** 같은 분류가 있어요. 보통은 **“글”** 을 씁니다.
- 목록에서 글을 누르면 **수정**, 위쪽 버튼으로 **새 글**을 만들어요.

## 3. 새 글 쓰기 — “New(＋)” 누르기

화면 위쪽의 **＋(New)** 버튼을 누르면 빈 글 작성 화면이 열려요.

## 4. 항목 채우기

위에서부터 차례로 채우면 돼요.

- **제목** — 글 제목.
- **slug (주소)** — 글 주소가 되는 값. **영어 소문자·숫자·하이픈(-)** 으로만! 예: `hello-world`. (한글로 쓰면 주소가 지저분해져요.)
- **게시일** — 자동으로 지금 시각이 들어가요. 여행 중 그때 시각으로 남기고 싶으면 바꿔도 돼요.
- **저자** — 현지 / 용배 중 선택.
- **카테고리** — 잡담 / 공부 / 같이 놀기 중 선택. **새 카테고리가 필요하면 용배(개발자)에게 문의**하세요(직접 추가는 안 돼요).
- **태그** — 여러 개면 **한 줄에 하나씩** 나눠서 적어요(항목 추가 버튼으로 하나씩).
- **요약** — 목록과 공유 미리보기에 나오는 짧은 소개(선택).
- **초안** — 새 글은 기본이 **초안(비공개)**. 다 쓰고 **체크를 해제**하면 그때 사이트에 공개돼요.

## 5. 본문 쓰기 — 알아두면 좋은 것들

본문은 대부분 **위지윅으로 자유롭게** 쓰면 되고, 아래 몇 가지만 정해진 방식이 있어요. (에디터엔 글자 그대로 보여도 **게시하면 예쁘게 바뀌니** 걱정 마세요.)

### 후리가나 · 병음 (한자 위에 읽는 법)

`{한자|읽기}` 형식으로 쓰면 글자 위에 작게 읽는 법이 올라가요.

- 일본어 — 입력 `{漢字|かんじ}` → 결과 {漢字|かんじ}
- 중국어 — 입력 `{中文|zhōngwén}` → 결과 {中文|zhōngwén}

가운데 세로 막대 `|` 는 키보드에서 **Shift + ₩(역슬래시 `\`) 키**로 입력해요.

### 이미지 넣기

에디터 툴바의 **이미지 버튼(Browse)** 으로 파일을 고르면 자동으로 붙고, 게시할 때 알아서 가볍게 최적화돼요. (끌어다 놓기도 되지만 Browse 가 가장 안정적이에요.)

### 코드 넣기 (기술 글)

코드는 백틱(``` ` ```) **세 개**로 위아래를 감싸면 색이 입혀지고, 마우스를 올리면 **복사 버튼**이 나와요. 첫 줄에 `title="파일이름"` 을 붙이면 파일명 탭도 생겨요.

```js title="예시.js"
const hello = "world";
console.log(hello);
```

문장 속 짧은 코드는 백틱 하나로 감싸요: 입력 `` `SELECT * FROM users` `` → `SELECT * FROM users`.

## 6. 게시하기

**초안 체크를 해제**하고 **저장**하면 끝! 실제 사이트에 반영되기까지 **1~2분** 걸려요. 그동안 관리자 화면 위쪽에 **진행 상태 표시(게시 중… → 게시 완료!)** 가 떠요. 완료 표시가 뜨면 사이트에서 확인하면 됩니다.
