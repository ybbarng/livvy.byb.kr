# 다국어 지원 로드맵

현지=일본어·중국어, 용배=프랑스어를 공부하며 기록합니다. "여러 언어를 예쁘게"는 주로 **타이포그래피** 문제입니다(사이트 UI 자체를 번역하는 i18n은 불필요 — UI는 한국어 유지). 복잡성을 피하려 단계적으로 진행합니다.

## 1단계 — 언어별 폰트·타이포 + 로딩 성능 ✅ (완료)

- fontsource 가변 폰트 self-host: **Noto Sans KR/JP/SC + 라틴**, Roboto(메타)
- `unicode-range`로 글자에 맞는 언어 폰트만 자동 다운로드
- `font-display: swap` — 폰트 로드 전 시스템 폰트로 즉시 표시(안 보이는 FOIT 없음)
- 프랑스어: 라틴 `latin-ext` subset이 발음기호(é, ç 등) 커버
- 한 글에 한/일/중/불이 섞여도 각 글자가 알맞은 폰트로 렌더

## 2단계 — 발음 표기 ✅ 완료

- 마크다운 문법 **`{한자|읽기}`** → `<ruby>` 자동 변환 (`plugins/remark-ruby.mjs`)
  - 후리가나: `{漢字|かんじ}` → 漢字 위에 かんじ
  - 병음: `{中文|zhōngwén}` → 中文 위에 zhōngwén
- 인라인/블록 코드 안은 변환하지 않음. 읽기(rt)는 테마 색(라이트/다크) 대응
- 파이프(`|`)가 있는 `{...}`만 변환 — 일반 중괄호는 그대로 둔다

## 3단계 — 글별 언어 태그·분류 (예정)

- frontmatter `lang` 필드 + 언어별 모아보기(예: 프랑스어 글만)

## 4단계 — 번역 보조 (예정)

- Sveltia 내장 **DeepL** 버튼으로 초안 작성 보조

## 폰트 로딩 성능 추가 최적화 (선택 · 낮은 우선순위)

`unicode-range`(글자에 맞는 subset만 자동 다운로드) + `font-display: optional`(CJK, 로딩 중 레이아웃이 안 바뀜 · 라틴은 swap)로 **실사용상의 문제(안 보임·밀림·스크롤 튐)는 이미 해결**됐다. 아래는 바이트를 더 줄이려는 순수 최적화라, 저트래픽 개인 블로그엔 이득이 크지 않다(급하지 않음):

- **빌드 타임 동적 subsetting**: 실제 페이지에 등장한 글자만 담은 폰트 생성(`subfont`/`fonttools` 계열) → CJK 폰트를 수 MB → 수십 KB로 축소
- 한국어 핵심 subset **preload**: `optional` 창(~100ms) 안에 폰트가 들어올 확률을 높여, 첫 방문에도 시스템 폰트 대신 Noto 를 보이게 함
